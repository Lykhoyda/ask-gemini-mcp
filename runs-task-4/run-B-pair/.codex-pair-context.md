# Project context for codex-pair (task 4)

This is a TypeScript shopping-cart library:

- `Cart` class with addItem / removeItem / setQuantity / applyDiscount / checkout
- Three discount types: PERCENTAGE, FIXED, BOGO (buy N get 1 free)
- Tax: 8% applied to subtotal-after-discount
- State machine: cart starts OPEN; checkout transitions to CHECKED_OUT; mutations after checkout MUST throw

**Stated requirements:**
- TypeScript strict mode, Zod input validation
- `tsc --noEmit` and `vitest run` must pass
- Mutations after checkout throw

**Threat / correctness surface to evaluate:**
- This is real shopping-cart code that would charge real users. Mistakes cost real money.
- Floating-point math on monetary values is incorrect in all currencies — integer minor units (cents) or a decimal library is the correct representation.
- "Buy N get 1 free" math is famously off-by-one. With buyN=2, every 3rd matching item is free (groups of N+1).
- Multiple discounts can stack — the order of application matters (percentage-then-fixed ≠ fixed-then-percentage), and there should be a documented precedence.
- State machine enforcement should be uniform across ALL mutating methods. Per-method inconsistency is a real bug.
- Input validation should reject implausible quantities (negative, NaN, zero where it doesn't make sense).
